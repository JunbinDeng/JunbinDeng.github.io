Java 基础 -- 语法（一）

Java基础语法将分成八个部分介绍：关键字，标识符，注释，常量与变量，运算符，语句，函数，数组。

1. 关键字

定义：被Java语言赋予了特殊含义的单词。

特点：关键字中所有字母都为小写。

作用于：

  • 定义数据类型：class, interface, byte, short, int, long, float, double, char, boolean, void；
  • 定义数据类型值：true, false, null；
  • 定义流程控制：if, else, switch, case, default, while, do, for, break, continue, return；
  • 定义访问权限修饰符：private, protected, public；
  • 定义类，函数，变量修饰符：abstract, final, static, synchronized；
  • 定义类与类之间关系：extends, implements；
  • 定义建立，引用及判断实例：new, this, super, instanceof；
  • 异常处理：try, catch, finally, throw, throws；
  • 包：package, import；
  • 其他修饰符：native, strictfp, transient, volatile, assert。

2. 标识符

定义：在程序中自定义的一些名称。

特点：由26个英文字母大小写，数字（0-9），符号（_$）组成。

合法标识符定义规则：

  • 数字不可以开头；
  • 不可以使用关键字。

提示：

  • Java严格区分大小写；
  • 为了提高阅读性，尽量起有意义的标识符名称。

多单词组合时的名称规范：

  • 包名：所有字母都小写。如：xxxyyyzzz；
  • 类名接口名：所有单词的首字母大写。如：XxxYyyZzz；
  • 变量名和函数名：第一个单词首字母小写，第二个单词开始每个单词首字母大写。如：xxxYyyZzz；
  • 常量名：多单词所有字母都大写，每个单词用下划线连接。如：XXX_YYY_ZZZ。

3. 注释

定义：用于注解说明程序的文字。

作用：提高代码的阅读性。

格式：

  • 单行注释
    • 格式：//注释文字
  • 多行注释
    • 格式：/*注释文字*/
  • 文档注释
    • 格式：/**注释文字*/

举例：

对于单行和多行注释，被注释的文字，不会被JVM（Java虚拟机）解释执行。

对于文档注释，是java特有的注释，其中注释内容可以被JDK提供的工具javadoc所解析，生成一套以网页文件形式体现的该程序的说明文档。

小贴士：
注释是一个程序员必须要具有的良好编程习惯。初学者编写程序可以养成先写注释再写代码的习惯，将自己的思想通过注释先整理出来，再用代码去体现。因为代码仅仅是思想的一种体现形式而已。
注释宜少而精，不宜多而滥。不写过于详细的注释，对显而易见的代码添加的注释，以及罗嗦的注释。

4. 常量与变量

4.1 常量

常量的定义：不能改变的数值。

常量的分类：

  • 整数常量：所有整数；
  • 小数常量：所有小数；
  • 布尔型常量：只有两个数值，true和false；
  • 字符常量：将一个数字，字母或者符号用一对单引号（’ ‘）标识；
  • 字符串常量：将一个或者多个字符用一对双引号（” “）标识；
  • null常量：只有一个数值就是null（空）。

4.1.1 进制

数运算的进位方法：

  • 二进制：0-1，满二进一；
  • 十进制：0-9，满十进一；
  • 八进制：0-7，满八进一；
  • 十六进制：0-9和A-F，满十六进一，用0x开头表示；

原码：所对应的正数二进制。

反码：所对应的正数二进制取反。

补码：所对应的正数二进制取反加1。是负数的二进制表现形式。

4.2 变量

变量的定义：内存中的一个存储区域。该区域有自己的名称（变量名）和类型（数据类型），该区域的数据可以在同一类型范围内不断变化。

作用：用来不断的存放同一类型的常量，并可以重复使用。

使用变量时注意：

  • 其作用范围（在一对{}之间有效）；
  • 初始化值。

定义变量的格式：

数据类型 变量名 = 初始化值;

小贴士：格式是固定的，记住格式，以不变应万变。

4.2.1 数据类型分类

注：Java语言是强类型语言，对于每一种数据都定义了明确的具体数据类型，在内存中分配了不同大小的内存空间。

  • 如果一个操作数是float型，计算结果就是float型；
  • 如果一个操作数是double型，计算结果就是double型。

6. 程序流程控制

程序流程控制包含：判断结构，选择结构，循环结构。

6.1 判断结构

6.1.1 if语句

格式1：

if (条件表达式) {
    执行语句;
}

格式2：

if (条件表达式) {
    执行语句;
} else {
    执行语句;
}

格式3：

if (条件表达式) {
    执行语句;
} else if (条件表达式) {
    执行语句;
......
} else {
    执行语句;
}

if语句特点：

  • 每一种格式都是单条语句。

  • 第二种格式与三元运算符的区别：

    • 三元运算符运算完要有值出现；
    • 好处是：可以写在其他表达式中。

  • 条件表达式无论写成什么样子，只看最终的结构是否是true或者false。

6.2 选择结构

6.2.1 switch语句

格式：

switch (表达式) {
    case 取值1:
        执行语句;
        break;
    case 取值2:
        执行语句;
        break;
    ......
    default:
        执行语句;
        break;
}

switch语句特点：

  • switch语句选择的类型只有四种：byte，short，int，char。
  • case之间与default没有顺序。先执行第一个case，没有匹配的case执行default。
  • 结束switch语句的两种情况：遇到break，或是执行到switch语句结束。
  • 如果匹配的case或者default没有对应的break，那么程序会继续向下执行，运行可以执行的语句，直到遇到break或者switch语句结束。

6.3 循环结构

循环结构的3个代表语句： while, do while, for。

6.3.1 while语句

格式：

while (条件表达式) {
    执行语句;
}

6.3.2 do while语句

格式：

do {
    执行语句;
} while (条件表达式);

do while特点是条件无论是否满足，循环体至少被执行一次。

6.3.3 for语句

格式：

for (初始化表达式; 循环条件表达式; 循环后的操作表达式) {
    执行语句;
}

注：

  • for里面的表达式运行的顺序：

    1. 初始化表达式，只读一次；
    2. 判断循环条件，为真就执行循环体；
    3. 执行循环后的操作表达式；
    4. 重复过程2到3，直到循环条件不满足为止。

  • while与for可以互换，区别在于，for为了循环而定义的变量在for循环结束后在内存中释放，而while循环使用的变量在循环结束后还可以继续使用。

  • 最简单无限循环格式：while(true) , for(;;)。无限循环存在的原因是并不知道循环多少次，而是根据某些条件，来控制循环。

6.4 其他流程控制语句

6.4.1 break语句

break（跳出）：结束整个循环体。

应用范围：选择结构和循环结构。

6.4.2 continue语句

continue（继续）：结束本次循环，继续下个循环。

应用范围：循环结构。

注：

  • 这两个语句离开应用范围，存在是没有意义的。
  • 这两个语句单独存在，下面都不可以有语句，因为执行不到。
  • 标号的出现，可以让这两个语句作用于指定的范围。

7. 函数

定义：函数就是定义在类中的具有特定功能的一段小程序。也称为方法。

格式：

修饰符 返回值类型 函数名(参数类型 形式参数1, 参数类型 形式参数2，...) {
    执行语句;
    return 返回值;
}
  • 返回值类型：函数运行后的结果的数据类型。
  • 参数类型：是形式参数的数据类型。
  • 形式参数：是一个变量，用于存储调用函数时传递给函数的实际参数。
  • 实际参数：传递给形式参数的具体数值。
  • return：用于结束函数。
  • 返回值：该值会返回给调用者。

特点：

  • 定义函数可以将功能代码进行封装，便于对该功能进行复用。
  • 函数只有被调用才会被执行。
  • 函数的出现提高了代码的复用性。
  • 当函数没有具体返回值时，返回值类型用关键字void表示，如果该函数中的return语句在最后一行可以省略不写。

注意：

  • 函数中只能调用函数，不可以在函数内部定义函数。
  • 定义函数时，函数的结果应该返回给调用者，交由调用者处理。

应用：

  • 明确要定义的功能最后的结果是什么；
  • 明确在定义该功能的过程中，是否需要未知内容参与运算。

Java 概述

1. Java 的简述

Java是1995年SUN（Stanford University Network）公司推出的一门高级编程语言。

Java是一门面向Internet的编程语言，目前已经成为Web应用程序的首选开发语言，具有简单易学，完全面向对象，安全可靠，跨平台等特点。目前，Java主要有三种技术架构：

  • J2EE（Java 2 Platform Enterprise Edition）企业版：是为开发企业环境下的应用程序提供的一套解决方案。该技术体系中包含的技术如Servlet Jsp等，主要针对于Web应用程序开发。
  • J2SE（Java 2 Platform Standard Edition）标准版：是为开发普通桌面和商务应用程序提供的解决方案。该技术体系是其他两者的基础，可以完成一些桌面应用程序的开发，比如Java版的扫雷、记事本等。
  • J2ME（Java 2 Platform Micro Edition）小型版：是为开发电子消费产品和嵌入式设备提供的解决方案。该技术体系主要应用于小型电子消费产品，如手机中的应用程序等。

在 5.0 版本后，这三种架构被更名为 JavaEE、JavaSE、JavaME。

2. Java 的跨平台性

通过Java语言编写的应用程序在不同的系统平台（Windows，Linux，Mac）上都可以运行。Java有一个自己独立的能解析Java软件的解析器，称之为Java虚拟机，即JVM（Java Virtual Machine）。只要在需要运行Java应用程序的操作系统上，先安装一个JVM就可以了，接下来就由JVM来负责Java程序在该系统中的运行。这样就实现了Java程序的跨平台性，也称为Java具有良好的可移植性。

高级语言的工作原理：

  • c 语言的编程过程：

    c源程序(扩展名.c) —> windows可执行性文件 —> windows操作系统(非跨平台)

  • Java 语言的编程过程：

    Java源程序(扩展名.java) —> Java字节码文件(扩展名.class) —> jvm虚拟机(跨平台)

3. Java 的环境搭建

3.1 介绍

JRE（Java Runtime Environment）是 Java 运行环境，包括 Java 虚拟机和 Java 程序所需的核心类库等。如果想要运行一个开发好的 Java 程序，计算机中只要安装 JRE 即可。

JDK（Java Development Kit）是 Java 开发工具包，提供给 Java 开发人员使用。其中包含了 Java 的开发工具，也包括了 JRE。所以安装了 JDK，就不用再单独安装 JRE 了。其中的开发工具包括：编译工具（javac.exe）和打包工具（jar.exe）等。

3.3 配置环境变量

3.3.1 Path 配置

每次执行java工具都要进入到其安装路径下的bin目录是非常麻烦的，我们希望在命令提示符中的任意目录下都可以执行java工具。所以，我们可以将java工具所在路径定义到path环境变量中，让系统帮我们去找运行执行的程序。这是一劳永逸的配置方式。

步骤：

我的电脑 —— 属性 —— 高级 —— 环境变量

编辑path环境变量，在变量值开始处加上java工具所在目录，如：C:\Program Files\Java\jdk1.8.0_25\bin，后面用;与其他值分隔开即可。

配置技巧：

如果不希望因为jdk的目录或者目录名称的改变，而不断更改path的值，从而导致对path变量值的误操作，可以通过以下技巧配置：

3.3.2 临时配置

临时配置方式只在当前命令提示符窗口有效，窗口关闭后配置则消失。这是为了避免在使用他人的电脑开发或者编译时，更改他人的环境变量从而给其造成不便。

方法：

通过set命令完成。

set: ：用于查看本机的所有环境变量信息。

set 变量名：查看具体一个环境变量的值。

set 变量名=：清空一个环境变量的值。

set 变量名=变量值：给指定变量定义具体值。

在原有环境变量值中加入新值的方法是，先通过%变量名%操作符获取到原有环境变量的值，加上新值后，再定义给该变量名，即：

set path=新值;%path%

3.3.3 Classpath配置

class文件是Java的运行文件，配置classpath可以实现在任意目录下都能执行某一目录下的class文件。

JVM查找类文件的顺序：

  1. 如果没有配置classpath环境变量，JVM只在当前目录下查找要运行的类文件。
  2. 如果配置了classpath环境，JVM会先在classpath环境变量值的目录中查找要运行的类文件。
  3. 值的结尾处如果加上分号，那么JVM在classpath目录下没有找到指定的类文件，会在当前目录下在查找一次。
  4. 值的结尾出如果没有分号，那么JVM在classpath目录下没有找到指定的类文件，不会在当前目录下查找。即使当前目录下有，也不会运行。

建议：

  • 配置classpath环境变量时，值的结尾处不要加分号。否则当JVM在classpath目录中没有找到指定的类文件的情况下，会运行当前目录下的类文件，运行结果容易让人产生困惑。
  • 如果需要先访问当前目录可以在classpath变量值的第一位添加“.”用来表示当前目录，然后再用“;”与其他值分隔开。

5. 注释

注释的作用：

解释说明程序，提高代码的阅读性。
调试程序。如果程序出现错误，可以利用注释来排错。

三种注释方式：

单行注释、多行注释、文档注释。

注释格式：

单行注释与多行注释的文字，不会被JVM解释执行。单数注释可以嵌套单行注释和多行注释，多行注释可以嵌套单行注释。但是，多行注释不能嵌套多行注释，因为多行注释的开头会和被嵌套的多行注释的结尾配对，导致后面的注释失效。

文档注释是Java特有的注释，通常书写在类、域、构造函数、方法、定义之前。注释内容可以被JDK中的工具javadoc.exe所解析，生成一套以网页文件形式体现的该程序的说明文档。
