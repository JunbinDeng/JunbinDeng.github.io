Java 概述

1. Java 的简述

Java是1995年SUN（Stanford University Network）公司推出的一门高级编程语言。

Java是一门面向Internet的编程语言，目前已经成为Web应用程序的首选开发语言，具有简单易学，完全面向对象，安全可靠，跨平台等特点。目前，Java主要有三种技术架构：

  • J2EE（Java 2 Platform Enterprise Edition）企业版：是为开发企业环境下的应用程序提供的一套解决方案。该技术体系中包含的技术如Servlet Jsp等，主要针对于Web应用程序开发。
  • J2SE（Java 2 Platform Standard Edition）标准版：是为开发普通桌面和商务应用程序提供的解决方案。该技术体系是其他两者的基础，可以完成一些桌面应用程序的开发，比如Java版的扫雷、记事本等。
  • J2ME（Java 2 Platform Micro Edition）小型版：是为开发电子消费产品和嵌入式设备提供的解决方案。该技术体系主要应用于小型电子消费产品，如手机中的应用程序等。

在 5.0 版本后，这三种架构被更名为 JavaEE、JavaSE、JavaME。

2. Java 的跨平台性

通过Java语言编写的应用程序在不同的系统平台（Windows，Linux，Mac）上都可以运行。Java有一个自己独立的能解析Java软件的解析器，称之为Java虚拟机，即JVM（Java Virtual Machine）。只要在需要运行Java应用程序的操作系统上，先安装一个JVM就可以了，接下来就由JVM来负责Java程序在该系统中的运行。这样就实现了Java程序的跨平台性，也称为Java具有良好的可移植性。

高级语言的工作原理：

  • c 语言的编程过程：

    c源程序(扩展名.c) —> windows可执行性文件 —> windows操作系统(非跨平台)

  • Java 语言的编程过程：

    Java源程序(扩展名.java) —> Java字节码文件(扩展名.class) —> jvm虚拟机(跨平台)

3. Java 的环境搭建

3.1 介绍

JRE（Java Runtime Environment）是 Java 运行环境，包括 Java 虚拟机和 Java 程序所需的核心类库等。如果想要运行一个开发好的 Java 程序，计算机中只要安装 JRE 即可。

JDK（Java Development Kit）是 Java 开发工具包，提供给 Java 开发人员使用。其中包含了 Java 的开发工具，也包括了 JRE。所以安装了 JDK，就不用再单独安装 JRE 了。其中的开发工具包括：编译工具（javac.exe）和打包工具（jar.exe）等。

其中的包含关系如下图：

简而言之，JDK 包含了 JRE，JRE 中包含了 JVM，使用 JDK 开发完成的 Java 程序，交给 JRE 去运行。

3.3 配置环境变量

3.3.1 Path 配置

每次执行java工具都要进入到其安装路径下的bin目录是非常麻烦的，我们希望在命令提示符中的任意目录下都可以执行java工具。所以，我们可以将java工具所在路径定义到path环境变量中，让系统帮我们去找运行执行的程序。这是一劳永逸的配置方式。

步骤：

我的电脑 —— 属性 —— 高级 —— 环境变量

编辑path环境变量，在变量值开始处加上java工具所在目录，如：C:\Program Files\Java\jdk1.8.0_25\bin，后面用;与其他值分隔开即可。

配置技巧：

如果不希望因为jdk的目录或者目录名称的改变，而不断更改path的值，从而导致对path变量值的误操作，可以通过以下技巧配置：

  1. 新创建一个环境变量JAVA_HOME记录jdk的目录。例如，变量名：JAVA_HOME，变量值：C:\Program Files\Java\jdk1.8.0_25。
  2. 更改path环境变量，加上%JAVA_HOME%\bin;

其中，%JAVA_HOME%的作用是可以动态获取名称为JAVA_HOME环境变量的值。

3.3.2 临时配置

临时配置方式只在当前命令提示符窗口有效，窗口关闭后配置则消失。这是为了避免在使用他人的电脑开发或者编译时，更改他人的环境变量从而给其造成不便。

方法：

通过set命令完成。

set: ：用于查看本机的所有环境变量信息。

set 变量名：查看具体一个环境变量的值。

set 变量名=：清空一个环境变量的值。

set 变量名=变量值：给指定变量定义具体值。

在原有环境变量值中加入新值的方法是，先通过%变量名%操作符获取到原有环境变量的值，加上新值后，再定义给该变量名，即：

set path=新值;%path%

3.3.3 Classpath配置

class文件是Java的运行文件，配置classpath可以实现在任意目录下都能执行某一目录下的class文件。

JVM查找类文件的顺序：

  1. 如果没有配置classpath环境变量，JVM只在当前目录下查找要运行的类文件。
  2. 如果配置了classpath环境，JVM会先在classpath环境变量值的目录中查找要运行的类文件。
  3. 值的结尾处如果加上分号，那么JVM在classpath目录下没有找到指定的类文件，会在当前目录下在查找一次。
  4. 值的结尾出如果没有分号，那么JVM在classpath目录下没有找到指定的类文件，不会在当前目录下查找。即使当前目录下有，也不会运行。

建议：

  • 配置classpath环境变量时，值的结尾处不要加分号。否则当JVM在classpath目录中没有找到指定的类文件的情况下，会运行当前目录下的类文件，运行结果容易让人产生困惑。
  • 如果需要先访问当前目录可以在classpath变量值的第一位添加“.”用来表示当前目录，然后再用“;”与其他值分隔开。

5. 注释

注释的作用：

解释说明程序，提高代码的阅读性。
调试程序。如果程序出现错误，可以利用注释来排错。

三种注释方式：

单行注释、多行注释、文档注释。

注释格式：

单行注释与多行注释的文字，不会被JVM解释执行。单数注释可以嵌套单行注释和多行注释，多行注释可以嵌套单行注释。但是，多行注释不能嵌套多行注释，因为多行注释的开头会和被嵌套的多行注释的结尾配对，导致后面的注释失效。

文档注释是Java特有的注释，通常书写在类、域、构造函数、方法、定义之前。注释内容可以被JDK中的工具javadoc.exe所解析，生成一套以网页文件形式体现的该程序的说明文档。
