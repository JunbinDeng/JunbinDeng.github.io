1. String类

字符串是一个特殊的对象，一旦初始化就不可以改变。

String str = “abc”;与String str1 = new String(“abc”);的区别在于：

1.1 String类的常用方法

1.1.1 获取

int length()：获取字符串的长度。
char charAt(int index)：通过索引获取对应的字符。

int indexOf(int ch)：返回ch在字符串中第一次出现的位置。
int indexOf(int ch, int fromIndex)：从fromIndex指定位置开始，获取ch在字符串中出现的位置。
int lastIndexOf(int ch)：从后往前找，返回ch在字符串中第一次出现的位置。
int lastIndexOf(int ch, int fromIndex)：从后往前找，从fromIndex指定位置开始，返回ch在字符串中第一次出现的位置。

int indexOf(String str)：返回str在字符串中第一次出现的位置。
int indexOf(String str, int fromIndex)：从fromIndex指定位置开始，获取str在字符串中出现的位置。
int lastIndexOf(String str)：从后往前找，返回ch在字符串中第一次出现的位置。
int lastIndexOf(String str, int fromIndex)：从后往前找，从fromIndex指定位置开始，返回ch在字符串中第一次出现的位置。

String substring(begin)：通过传入的索引值开始向后截取，截取的是索引到length。
String substring(begin, end)：通过传入的两个索引值截取，包含头，不包含尾。

1.1.2 判断

boolean contains(str)：字符串中是否包含某一子串。
boolean isEmpty()：原理就是判断长度是否为0。

boolean startsWith(str)：字符串是否是以指定内容开头。
boolean endsWith(str)：字符串是否是以指定内容结尾。

boolean equals(str)：判断字符串内容是否相同。复写了Object类中的equals方法。
boolean equalsIgnoreCase()：判断内容是否相同，并忽略大小写。

1.1.3 转换

String(char[])：将字符数组转成字符串。用于构造函数。
String(char[], offset, count)：将字符数组转成字符串，从offet开始，截取count个。用于构造函数。

static String copyValueOf(char[])：将字符数组转成字符串。用于静态方法。
static String copyValueOf(char[] data, int offset, int count)：将字符数组转成字符串，从offet开始，截取count个。用于静态方法。
static String ValueOf(char[])：将字符数组转成字符串。用于静态方法。
static String ValueOf(char[] data, int offset, int count)：将字符数组转成字符串，从offet开始，截取count个。用于静态方法。

char[] toCharArray()：将字符串转换成字符数组。

String(byte[])：将字节数组转换成字符串。
String(byte[], offset, count)：将字节数组转成字符串，从offet开始，截取count个。

byte[] getBytes()：将字符串转成字节数组。括号内可以指定编码表。

static String valueOf(int)：将int型转成字符串。
static String valueOf(double)：将double型转成字符串。

String toUpperCase()：将字符串转成大写。
String toLowerCase()：将字符串转成小写。

1.1.4 替换

String replace(oldchar, newchar)：把旧值替换成新值。

1.1.5 切割

String[] split(String reg)：通过regex切割字符串，切割后会产生一个字符串数组。

1.1.6 比较

int compareTo(String anotherString)：比较两个字符串的自然顺序。

小贴士

  • indexOf(str)可以索引str第一次出现的位置。如果返回-1，表示该str不在字符串中存在。所以，既可以用于判断是否包含指定字符串，又可以获取出现的位置。例如：
    if (str.indexOf(“abc”) != -1) {}

2. StringBuffer

StringBuffer是字符串缓冲区，字符串的组成是通过该类实现的。

特点：

  • 是一个容器。
  • 长度可以变化。
  • 可以操作多个数据类型。
  • 最终会通过toString方法变成字符串。

2.1 StringBuffer特有方法

StringBuffer append(int x)：将制定数据作为参数添加到已有数据结尾处。

StringBuffer insert(int index, String str)：可以将数据插入到指定index位置。

StringBuffer delete(int start, int end)：删除缓冲区中的字符，包含start，不包含end。

StringBuffer deleteCharAt(index)：删除指定位置的字符，

StringBuffer reverse()：反转。

JDK1.5出现一个StringBuilder，区别是StringBuffer是同步的，StringBuilder是非同步的。

4. 泛型

泛型是JDK1.5以后出现的类型安全机制，用于解决安全问题。

在JDK1.5之前，没有泛型的情况的下，通过对类型Object的引用来实现参数的“任意化”，“任意化”带来的缺点是要做显式的强制类型转换，而这种转换是要求开发者对实际参数类型可以预知的情况下进行的。对于强制类型转换错误的情况，编译器可能不提示错误，在运行的时候才出现异常，这是一个安全隐患。

泛型格式：通过<>来定义要操作的引用数据类型。

泛型的特点：

  • 提高了程序的安全性。
  • 将运行期遇到的问题ClassCastException转移到了编译期。
  • 省去了类型强转的麻烦。
  • 泛型类的出现优化了程序设计。

自定义泛型类：当类中要操作的引用数据类型不确定的时候，需要自定义泛型类。
