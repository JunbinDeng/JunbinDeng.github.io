1 多线程的概念

要知道什么是多线程，先要知道什么是线程。要知道什么是线程，先要知道什么是进程。

进程：一个正在执行中的程序。每一个进程的执行都有一个执行顺序，该顺序是一个执行路径，或者叫一个控制单元。

线程：进程中的一个独立的控制单元。线程在控制着进程的执行。一个进程中至少有一个线程。

多线程：JVM启动的时候会有一个进程java.exe，该进程中至少有一个线程负责java程序的运行，而且这个线程运行的代码存在于main方法中。该线程称之为主线程。JVM不止启动一个线程，还有负责垃圾回收机制的线程。像这样一个进程有多个线程的执行方式称为多线程。

多线程存在的意义：多线程能让程序产生同时运行的效果，提高执行效率。以JVM为例，主线程的代码执行完后会留在堆内存中，成为垃圾。启用负责垃圾回收机制的线程，可以释放堆内存中的垃圾，让程序执行更高效。

多线程的特性：随机性。当多个线程都在获取cpu的执行权时，cpu在多个线程之间做快速切换。在某一个时刻，只运行一个程序（多核除外）。

1.1 线程的状态

线程的状态：

  • 被创建
  • 运行
  • 冻结：放弃了执行资格
  • 临时：具备了执行资格，但不具备执行权
  • 消亡

2.2 实现Runnable接口

步骤：

  1. 子类覆盖接口中的run方法。
  2. 通过Thread类创建线程，并将实现了Runnable接口的子类对象作为参数传递给Thread类的构造函数。
  3. Thread类对象调用start方法开启线程。

Runnable的子类对象传递给Thread类的构造函数的原因：自定义run方法所属的对象是Runnable接口的子类对象，所以要让线程去指定对象的run方法，就必须明确该run方法所属对象。

实现方式和继承方式的区别：

  • 实现方式：线程代码存放在接口的子类run方法中。
  • 继承方式：线程代码存放在Thread子类run方法中。

实现方式避免了单继承的局限性。建议使用实现方式。

3 线程安全问题

当多条语句在操作同一个线程共享数据时，一个线程的多条语句还没执行完，另一个线程参与进来执行，导致共享数据的错误。

导致安全问题出现的原因：

  • 多个线程访问出现延迟。
  • 线程随机性。

解决办法：对多条操作共享数据的语句，只能让一个线程先执行完。在执行过程中，其他线程不可以参与执行。

4.1 同步的特点

同步的前提：

  • 同步需要两个或者两个以上的线程。
  • 多个线程使用的是同一个锁。

同步的弊端：当线程相当多时，每个线程都会去判断同步上的锁，耗费资源，降低了程序的运行效率。

4.2 多线程下的单例设计模式

饿汉式不存在安全问题，因为没有多个线程共同操作数据的情况。

懒汉式存在线程安全问题，可以加入synchronized来解决。

这样的方式比直接在Single函数上加入synchronized效率更高。这样的方式下，如果Single对象已创建，则直接获取即可，而不用判断是否能够获取锁。如果当前线程发现Single对象尚未创建，才需要判断是否能够获取锁。但是，如果直接使用同步函数，无论Single对象有没有创建，都需要先判断是否能够获取锁，效率就不高了。

5 线程间通信

线程间通信是指等待/唤醒机制。

方法：

  • wait()：让线程处于冻结状态，被冻结的线程会存储到线程池中。
  • notify()：唤醒线程池中的任意一个线程。
  • notifyAll：唤醒线程池中的所有线程。

关于多生产者-多消费者程序类型问题的解释：

  • 当多个生产者消费者出现时，通过while让获取执行权的线程判断标记。
  • 需要将对方的线程唤醒时，仅仅用notify是不可以的。因为有可能出现只唤醒本方的情况，导致所有线程都等待。所以可以通过notifyAll的形式来完成。
  • 需要只唤醒对象，要使用Lock接口替代synchronized。

操作线程的方法wait()，notify()，notifyAll()定义在Object类中的原因：

  • 这些方法存在于同步中。
  • 使用这些方法时必须要标识所属的同步的锁。
  • 锁可以使任意对象，所以任意对象调用的方法一定定义在Object类中。

wait()和sleep()的区别：

  • wait()：释放cpu执行权，释放锁。
  • sleep()：释放cpu执行权，不释放锁。

5.1 优化的等待唤醒机制

JDK1.5版本提供了一些新的对象，优化了等待唤醒机制。

  • 将synchronized替换成了Lock接口。将隐式锁升级成了显示锁。
    • 获取锁：lock();
    • 释放锁：unlock(); 释放的动作一定要执行，通常定义在finally中。
    • 获取Condition对象：newCondition();
  • 将Object中的wait，notify，notifyAll方法都替换成了Condition的await，signal，signalAll。
    • 不同之处在于，一个同步代码块具备一个锁，该锁具备自己独立的wait和notify方法。
    • wait，notify等方法被封装进特有对象Condition，而一个Lock锁可以有多个Condition对象。

5.2 停止线程

停止线程的方法：定义循环结束标记。线程运行代码一般都是循环，只要控制了循环即可。

interrupt（中断）方法：该方法是结束线程的冻结状态，使线程回到运行状态中来。

5.3 线程的其他方法

setPriority(int num)：设置优先级。

  • MAX_PRIORITY：最高优先级10。
  • MIN_PRIORITY：最低优先级1。
  • NORM_PRIORITY：分配给线程的默认优先级。

setDaemon(boolean)：将线程标记为后台线程。后台线程和前台线程一样会抢执行权，区别在于，当前台线程都运行结束后，后台线程会自动结束。

join()：等待线程结束。当A线程执行到了B的join方法时，A就会处于冻结状态。当B运行结束后，A就会具备运行资格，继续运行。

yield：临时暂停，可以让线程释放执行权。
